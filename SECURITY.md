# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please report it privately to project maintainers before opening a public issue.

Include:

- affected files/components
- reproduction steps
- impact assessment
- suggested remediation if available

## Response Expectations

Maintainers will acknowledge reports, triage severity, and coordinate remediation.

## Scope

This includes vulnerabilities in:

- tooling scripts
- CI/workflow configurations
- example/reference execution paths
- schema validation and ingestion pipelines
